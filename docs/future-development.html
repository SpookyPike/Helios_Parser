What is already built

  • explicit edge/center coordinate semantics across parser, HDF5, runtime, viewer, and derived layers
  • run-status metadata and damaged-final-block handling
  • viewer/shared raw-data cache publication into derived mode
  • bounded caches with lightweight observability
  • persistent background task execution with cooperative cancellation and latest-wins apply
  • fast legacy Shock path
  • lazy and cached WaveFront branch tracking
  • interface-aware event diagnostics
  • separate Preheat diagnostics tab with ROI override and target-entry logic
  • module-local XRD / Plasmon / Transmission / Spectroscopy controls

Likely next priorities

Wave physics refinement

Improve branch evidence formulas, fitting, and association quality on more layered and reflected/release-heavy cases without breaking the current stable quick-look UX.

WaveFront interpretation depth

Add stronger summaries and comparison tools around branch history, ambiguity, and advanced event interpretation while keeping the tab readable.

Preheat follow-up

Refine target heuristics, add more design-oriented summaries, and improve how multiple candidate target regions are compared.

Distribution polish

Move from source-first bundle distribution toward a more polished packaged desktop release once dependencies and examples stabilize further.

Major physics and diagnostics still remaining

  • deeper wave/interface interpretation beyond current quick-look event summaries
  • stronger validation-style summaries such as RH residuals, Mach-like proxies, and impedance interpretation where scientifically justified
  • more advanced preheat source/budget interpretation and confidence framing
  • fuller treatment of cylindrical downstream transport physics in modules that are currently only geometry-corrected at the weighting/path level
  • richer VISAR-aware diagnostics if future workflows require it
The current 1D-based interpretation remains useful, but it is still a quick-look interpretation layer. Documentation and UI must continue to state that honestly.

Technical debt and known limitations

  • some large advanced datasets still take noticeable time on first WaveFront/Preheat load
  • the release bundle is source-based rather than a frozen binary package
  • bundle and docs validation are practical rather than fully automated site/build validation
  • the test tree contains many phase-named suites that are useful historically but not yet reorganized into a cleaner long-term taxonomy

Performance improvement opportunities

  • trim remaining heavy advanced first-load costs on large layered runs
  • continue reducing redundant UI-side rerender work after selector changes
  • improve bundle-generation and screenshot-refresh automation
  • expand instrumentation summaries that help maintainers spot cache/invalidation regressions faster

UI and workflow opportunities

  • continue polishing advanced-tab readability without turning the UI into a developer-only dashboard
  • make more derived interpretation examples directly discoverable from the UI and docs
  • improve release-bundle onboarding with a future binary or installer path
  • keep combo/scroll/splitter interaction stable across dense scientific panels

Current interpretation boundaries

The application is strongest today as a fast, trustworthy, scientifically honest quick-look environment with an increasingly rich advanced branch-analysis layer. It is not yet a final all-physics publication platform, and the roadmap should preserve that honesty instead of masking it.
