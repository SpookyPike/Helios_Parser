Release Notes

HELIOS Parse / View 1.1.0, released 2026-05-11. Code developed by Dmitrii Bespalov at European XFEL.

Documentation Index User Guide Developer Guide README

Why 1.1.0

This release finalizes the production parser/reader pipeline for BPF-primary HELIOS data while preserving LOG-only workflows and the existing physics-gated desktop behavior.

Highlights in this release

  • production parser accepts `.log` and `.bpf` inputs with explicit BPF-primary routing when a companion BPF exists
  • HDF5/H5D schema version 2.0 stores self-describing field metadata, source provenance, axis semantics, and aliases
  • BPF/LOG overlap fields were systematically cross-validated, including radiation pressure/energy and cumulative laser source behavior
  • remaining unresolved BPF records are preserved with honest raw names, confidence/status metadata, and forensic documentation
  • release archive excludes tests, legacy shadow modules, reports, caches, and development audit outputs

Changelog

1.1.0

  • added production BPF parser integration and BPF-primary HDF5 conversion
  • added transparent `.log` / `.bpf` input routing, including LOG+BPF, LOG-only, and BPF-only modes
  • added dynamic reader field discovery and schema-aware plotting-mode selection
  • validated BPF record 14 as padded radiation energy density and derived LOG-compatible radiation pressure/energy aliases
  • documented final BPF field-forensics outcomes and confidence levels

1.0.1

  • added backend feature gating for production-hidden physics modules
  • added explicit HDF5 snapshot and trace accessors for targeted viewer reads
  • vectorized the spectroscopy thermal-width time-series calculation
  • cleaned release packaging and added bundle-content validation

1.0.0

  • production GUI hides Plasmon/XRTS and Transmission unless HELIOS_DEV_MODE=1 or HELIOS_ENABLE_EXPERIMENTAL=1 is set
  • kept production-visible Shock, WaveFront, Preheat, XRD, and Spectroscopy workflows with lightweight physical sanity checks
  • rebuilt release metadata and examples for the 1.0 line

0.9.1

  • documented model-aware Transmission with explicit mode selection and selected-result traces
  • documented the XCOM seam, canonical material resolution from parsed EOS/opacity metadata, and precomputed cold-XCOM fallback-table behavior
  • documented expanded plasmon/XRTS quick-look modes and warning semantics for degenerate, non-collective, or out-of-domain states
  • updated bundle-facing quick-start text and release metadata
  • rebuilt the release archive from the current repository state

0.9.0

  • major documentation expansion for users, developers, maintainers, and collaborators
  • release metadata and attribution added to About text, README, docs, and package metadata
  • shareable Windows/Linux source bundle with launch/bootstrap scripts and included demo HDF5 runs
  • current WaveFront, interface-event, and Preheat behavior documented with screenshots and interpretation notes

Bundle contents

  • source tree under src/
  • updated documentation under docs/
  • selected packaging script under scripts/
  • Windows launcher: Run_HELIOS_Analyzer.ps1
  • Linux launcher: run_helios_analyzer.sh
  • top-level quick start: README_FIRST.md
  • selected demo HDF5 examples under examples/

The 1.1.0 bundle intentionally omits tests, reports, old release outputs, and legacy shadow source files. Experimental Transmission/XCOM support remains in source behind development flags.
